# Collaboration

这里说明如何与 VIRTURA-Collective 协作。

## Collaboration Context

VIRTURA 的协作方式以项目制、研究制与跨角色协作为主。

它更像一个持续生成中的创作共同体，而不是固定岗位式组织。

## Collaboration Principles

这部分来自旧团队仓，但已经收束成适合公开使用的版本：

1. 作品优先：协作最终应回到真实作品、真实研究或真实发布。
2. 资产沉淀：一次工作，尽量形成可复用的文档、素材、流程或模块。
3. 轻量协同：优先用清晰分工和低摩擦流程推进，不做过度管理。
4. 工具服务创作：技术和自动化是支撑层，不反过来夺走创作主线。
5. 边界清楚：公开内容、私有内容、实验内容要分层。

## Typical Collaboration Directions

- visual and stage design
- digital performance
- immersive spatial projects
- research and writing
- technical prototyping
- documentation and archiving

## Participation Modes

目前更适合的参与方式有三种：

### 1. Project-based

围绕单个作品、展演或专题短期协作。

### 2. Research-based

围绕方法论、文档系统、项目复盘和知识整理参与。

### 3. Long-term

在作品、研究、发布和系统建设之间持续协作。

## Lightweight Onboarding

如果你准备加入某个方向，建议先准备 4 项最基本信息：

1. 你关注的方向
2. 你可投入的时间
3. 你已有的相关经验
4. 你更适合项目协作、研究协作还是长期协作

## First Contribution Suggestions

第一次参与时，更适合从小而清楚的任务开始：

- 帮某个作品补文档
- 帮某个项目整理外部资料
- 参与一篇研究或复盘的编校
- 参与小型技术验证或展示页搭建

# Activities

这里整理 `VIRTURA` 当前对外开放的活动、展映、测试与公开说明。

如果你是从小红书、朋友转发、活动海报或某一页 GitHub 链接来到这里，这页应该先回答一件很简单的事：

最近到底有什么真的在发生，我应该先从哪里跟。

## 推荐先做的一件事

如果你想稳定地跟着我们的公开活动走，最直接的方式是先订阅：

- [VIRTURA 官方活动日历](https://lu.ma/VIRTURA)

这是我们当前最重要的公共活动入口。新的茶话会、线上演出空间测试、展映、公开说明和观看入口开放，都会优先从这里继续发布。

## 我们现在主要做哪几类活动

目前公开活动大致分成 5 类：

1. `Space Salon / 数字沙龙 / 交流客厅`
   更偏社群交流、嘉宾分享、一起放东西看、一起聊最近在做什么。
2. `太空气球 / Baloon Live Space`
   更偏线上 live set、音画实验、演出空间测试和共测夜。
3. `MOVA / Museum of Virtual Art`
   更偏线上展映、作品观看入口和轻量数字展览。
4. `在制内容审计会`
   更偏把正在做的页面、视觉、声音、空间内容拿出来，一起挑问题。
5. `Digital Library / Digital Stage 公开说明`
   更偏围绕仓库、归档、发布结构和 AI 时代的内容管理，做短而实用的公开说明。

另外还会穿插一些 `观看入口开放`、`页面上新`、`公开测试` 这类更轻的节点。

## 为什么我们持续做公共活动

这些活动对我们来说，不只是宣传动作。

它更像一个长期开放的公共前台。有人会从这里认识团队，有人会只是来旁听一次，有人是想看看最近有什么新作品，有人则是想慢慢判断自己是不是愿意继续交流或合作。

所以这里不会默认每个人都要加入我们。你也完全可以只是订阅日历，偶尔来一场，看看最近在发生什么。

更偏线上展映、作品观看室、轻量展览，以及从 GitHub library 快速整理成公共观看入口的发布动作。

当前入口：

更偏把正在做的页面、视觉、声音、空间内容和版本拿出来，一起看问题出在哪里。

更偏围绕仓库、归档、版本、页面前台和 AI 时代的内容管理，做短而实用的公开说明。

重点不是卖概念，而是把一套确实能继续工作的组织方法讲明白。

## 日历如何和 GitHub 配合

我们会尽量把日历和 GitHub 做成一套自然的前后衔接：

- `Luma` 负责订阅、报名、提醒和公开发生的那个时刻
- `VIRTURA-Collective` 负责团队、作品、活动和合作入口
- `VIRTURA-SpacePort` 负责知识网络、项目档案、活动归档和 `MOVA` 线上展映入口

所以一个节点可以不是“来现场见面”，也可以是：

- 一个线上展映正式开启
- 一个观看入口开放
- 一个公开测试开放申请
- 一次围绕归档与发布结构的公开说明

## 为什么这里不是一堆独立活动页面

`VIRTURA-Collective/activities` 现在不再试图把每一期活动都拆成很多分散页面。

现在更明确的分工是：

- 这里负责总入口、时间线、计划中节点和继续进入的方向
- `VIRTURA-SpacePort/stations` 负责更完整的活动站点、系列归档和发生现场
- `portfolio/workshops` 负责一些已经发生过的公开工作坊和历史条目

所以这里看起来比以前更干净，不是因为内容没了，而是因为过去分散的活动内容被收回到更适合长期承接它们的位置了。

## 人通常怎么靠近这个网络

很多关系都不是从正式“加入申请”开始，而是从这些活动场景进入：

1. 先旁听一次公开活动或线上分享
2. 再对某个系列、某个研究方向或某个小模块产生兴趣
3. 最后根据节奏，决定是继续关注、参与一次具体协作，还是长期保持联系

这也是为什么活动页很重要：它不只服务于合作，也服务于长期关注这个网络的人。

## 如果你想进一步参与

目前更适合的进入方式有 4 种：

1. 围绕某个项目系列参与一次清晰的协作
2. 围绕研究、写作或知识网络贡献一个小模块
3. 围绕工具、模板或发布结构参与搭建
4. 先从活动、沙龙和公开交流建立连接

更具体的参与说明见：[Collaboration](../collaboration/README.md)

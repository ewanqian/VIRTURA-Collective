# VIRTURA 未来 8 周活动路线图 2026-03-16

这份文档直接服务于 3 件事：

- 给 [lu.ma/VIRTURA](https://lu.ma/VIRTURA) 提供未来 1-2 个月的真实排期
- 让 `VIRTURA-Collective/activities` 不只是活动说明，而是一条真的会往前走的时间线
- 让 `VIRTURA-SpacePort/stations` 里的活动站点、展映入口和知识网络，能按节奏被反复激活

它不是纯脑暴，而是一份偏执行的工作稿。

---

## 一、现在这套活动机制怎么分

这一轮排期不再用太空、太满、太像说明书的名字去撑场面。

目前更适合对外使用的，是下面这套分法：

### 3 条长期活动线

- `Space Salon / 数字沙龙 / 交流客厅`
- `太空气球 / Baloon Live Space`
- `MOVA / Museum of Virtual Art`

### 3 类配套节点

- `在制内容审计会`
- `Digital Library / Digital Stage 公开说明`
- `发布与观看入口开放`

这样分的原因很直接：

1. `茶话会` 负责社群交流、嘉宾分享、一起放内容看
2. `太空气球` 负责线上演出空间、音画实验和共测
3. `MOVA` 负责展映、数字展览和观看入口
4. `审计会` 负责认真挑问题，不假装一团和气
5. `公开说明` 负责把仓库、归档、页面和 AI 时代的内容管理讲清楚
6. `入口开放` 负责把具体页面、观看室和版本入口正式接进日历

---

## 二、这 8 周的总体节奏

这 8 周不追求“每场都很重”，而追求节奏稳定、每一场都说得清。

大致节奏可以这样理解：

- 周五晚上：更适合 `MOVA`、`公开说明`、`观看入口开放`
- 周六晚上：更适合 `Space Salon`、`太空气球`、`在制内容审计会`

关键判断是：

- `Space Salon` 不做团队说明会，而做交流客厅
- `公开说明` 不卖概念，而讲能落地的内容结构
- `在制内容审计会` 不互相捧场，而一起挑问题
- `MOVA` 不追求大而全，而先把“从库到展”跑通

## 五、每场活动为什么这样排

### 1. 先把“真的在发生”做出来

`2026-03-21` 到 `2026-04-04` 这段，不要急着讲完整体系。

更重要的是让外部感知到：

- 你们真的在测试
- 你们真的在交流
- 你们真的在发布
- 你们也真的愿意把还没做好的东西拿出来接受批评

所以这段的主线是：

- `Alpha`
- `Vol.3`
- `MOVA 展映 01`
- `在制内容审计会 01`

### 2. 再把“怎么整理、怎么继续工作”讲出来

`2026-04-10` 到 `2026-04-25` 这段，更适合把内容管理、归档结构和发布机制往前推。

原因是外部已经先见到发生中的节点，这时再讲：

- `Digital Library / Digital Stage`
- `知识网络怎么变成别人真能用的入口`
- `观看入口开放`
- `茶话会里的共享观看`

会比一上来就讲抽象主张更容易被接受。

### 3. 最后把“展映与第二轮测试”拉通

`2026-05-01` 以后更适合做：

- 第二个线上主题小展
- 第二次审计会
- 第二轮公开说明
- 第二轮 Beta 共测

因为这时第一波公共线已经形成回访理由，适合把人从“来一次”变成“继续跟着走”。

## 七、每场最少需要准备什么

为了防止排期看起来很多、实际落不下去，建议每类节点都只要求最少准备量。

| 节点类型 | 最少准备项 |
|---|---|
| `Space Salon` | 一个主题、一位主分享人或一组带来的内容、一条后续阅读入口 |
| `太空气球` | 一个测试目标、一段可播放内容、一种反馈方式 |
| `在制内容审计会` | 一个明确问题、一种带材料的方式、一条会后继续交流的入口 |
| `公开说明` | 一页核心文档、一个具体案例、一次 20-40 分钟的结构 |
| `发布与观看入口开放` | 一个真的能看的页面、一个明确开放时刻 |
| `MOVA` | 一条观看路线、一个开幕时刻、2-4 个入口页面 |

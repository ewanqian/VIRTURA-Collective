## Core Definition

VIRTURA 是一个去中心化的艺术协作网络。

它不是行政化的固定编制团队，而是围绕作品、研究、演出、空间实验与数字舞台持续协作的创作共同体。

加入这个网络，也不意味着个人表达必须让位给"团队统一口径"。很多成员保留自己的个人项目、个人研究和个人发布节奏；团队更像是把这些方向在合适的时候接成协作，而不是把每个人都压成同一种身份。

## How The Network Works

这个网络的运行方式更接近“多层协作”，而不是单一中心控制:

| Layer | What It Does | Typical Output |
|---|---|---|
| core collaborators | 持续维护团队方向、项目主线与公开入口 | 系列作品、关键决策、对外结构 |
| project cells | 围绕单个作品、展演、展览或研究专题展开协作 | 版本、演出、展览、短期实验 |
| research and writing contributors | 参与方法整理、复盘、理论与知识编辑 | 文章、索引、知识网络条目 |
| tools and systems contributors | 参与工具验证、档案结构、发布流程与自动化 | 原型、模板、工作流、前台结构 |

这意味着 VIRTURA 可以同时容纳长期伙伴、项目制协作者、研究参与者和临时加入的支持角色，而不需要把所有人压进同一种组织关系。

它的优势很明确：

- 可以让不同人的个人方向继续生长
- 可以在需要时快速形成项目协作
- 不用先搭一层很重的组织壳，才能开始做事

它的代价也很明确：

- 节奏不一定总是整齐
- 不是每次公开招募都会立刻收到回应
- 需要更依赖真实项目和真实活动来维系关系

## What We Actually Do

当前最接近 VIRTURA 真实工作的几条线是：

- 数字舞台与在线发布

## Working Model

VIRTURA 更接近项目制协作网络，而不是固定岗位式组织。

它允许：

- 核心成员的长期协作
- 围绕单个项目的临时加入
- 艺术、技术、研究、文档等多角色并行
- 团队本体、公共前台、发布出口、工具工坊分别存在

## Activities And Rhythm

VIRTURA 的协作并不只发生在项目交付时，也发生在持续的公共活动与内部整理中。

当前已经能公开识别的活动节奏包括:

- 线下沙龙与分享
- 工作坊与公开活动
- 展演与展览现场
- 线上直播、远程讨论与阶段复盘

这些活动一方面服务于公众理解作品，另一方面也为未来的协作者提供进入网络的真实场景。

入口见: [Activities](../activities/README.md)

## Core Values

这部分从旧团队仓中提炼，只保留真正长期有用的部分：

1. 工程化思维：系统化、可复用、可迭代。
2. 极简原则：先抓主要矛盾，不堆伪完整结构。
3. 资产沉淀：一次创作，应尽可能转化为可复用资产。
4. 去中心化：角色可以流动，贡献不只来自固定岗位。
5. 虚实共生：像对待真实空间一样对待数字空间。

## Why This Repository Exists

过去团队介绍、作品入口、组织说明、研究方向和导航关系分散在不同仓库中，外部访客很难快速理解：

- 团队本体是什么
- `SpacePort` 是什么
- 哪些内容是作品
- 哪些内容是研究或工具

这个仓库的存在，就是为了把“团队本体”稳定下来。

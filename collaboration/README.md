# Collaboration

这里说明如何与 VIRTURA-Collective 协作。

我们更希望这页像“怎么开始一起做事”的入口，而不是一份空泛的合作口号。

## Collaboration Context

VIRTURA 的协作方式以项目制、研究制与跨角色协作为主。

它更像一个持续生成中的创作共同体，而不是固定岗位式组织。

## 这页主要给两类人看

- 有项目、场地、预算、资源或发布场景，想找合适团队协作的人
- 正在成长中的创作者、研究者、整理者，想从一个具体方向开始参与的人

## Collaboration Principles

这部分来自旧团队仓，但已经收束成适合公开使用的版本：

1. 作品优先：协作最终应回到真实作品、真实研究或真实发布。
2. 资产沉淀：一次工作，尽量形成可复用的文档、素材、流程或模块。
3. 轻量协同：优先用清晰分工和低摩擦流程推进，不做过度管理。
4. 工具服务创作：技术和自动化是支撑层，不反过来夺走创作主线。
5. 边界清楚：公开内容、私有内容、实验内容要分层。
6. 个人表达保留：成员可以保留自己的个人项目、个人研究和个人发布口径，团队协作不覆盖个人作者性。

## Typical Collaboration Directions

- visual and stage design
- digital performance
- spatial and exhibition projects
- research and writing
- technical prototyping
- documentation and archiving

## Build Tracks

如果你想参与“建造”而不仅是围观，目前最清楚的 4 条参与路径是:

1. 作品与展演: 进入某个项目系列，参与内容、空间、视觉、声音或现场版本
2. 研究与写作: 参与理论梳理、案例研究、复盘、编辑和公开文章
3. 工具与系统: 参与 `SceneForge`、模板、前台结构、发布流程与工作流实验
4. 知识网络: 帮助整理学习路径、案例索引、方法框架与术语入口

其中第 3 和第 4 条不要求你先成为核心成员，更适合从边界清晰的小任务开始。

## Participation Modes

目前更适合的参与方式有三种：

### 1. Project-based

围绕单个作品、展演或专题短期协作。

### 2. Research-based

围绕方法论、文档系统、项目复盘和知识整理参与。

### 3. Long-term

在作品、研究、发布和系统建设之间持续协作。

## Lightweight Onboarding

如果你准备加入某个方向，建议先准备 4 项最基本信息：

1. 你关注的方向
2. 你可投入的时间
3. 你已有的相关经验
4. 你更适合项目协作、研究协作还是长期协作

如果你希望参与工具或知识网络的建设，再额外补两项：

5. 你更愿意做内容整理、结构设计、前端页面、文档规范还是流程自动化
6. 你能否接受先从一个小而可交付的模块开始

## First Contribution Suggestions

第一次参与时，更适合从小而清楚的任务开始：

- 帮某个作品补文档
- 帮某个项目整理外部资料
- 参与一篇研究或复盘的编校
- 参与小型技术验证或展示页搭建
- 帮 `knowledge-network` 整理一个主题入口
- 帮某个系列补 `manifest`、公开资料页或活动时间线

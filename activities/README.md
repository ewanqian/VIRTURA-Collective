# Activities

这里整理 VIRTURA 的公共活动、沙龙、工作坊与线上线下时间线。

它和 [Performances](../performances/README.md) 的关系很近，但重点不同:

- `Activities` 更关注团队如何与公众、同行和未来协作者发生连接
- `Performances` 更关注作品在具体场域中的呈现

## Read This Page For

- 团队是否在持续发生真实活动
- 人通常怎样通过活动进入协作网络
- 哪些节点已经可以作为公开时间线阅读

## Why This Matters

如果一个团队只有作品列表，没有活动与时间线，外部就很难判断它是否真的在持续运转。

这一页存在的目的，是把 VIRTURA 作为一个去中心化协作网络的“日常发生机制”展示出来:

- 沙龙和公开分享
- 工作坊与方法传播
- 线上讨论与阶段性直播
- 与展演、展览相互连接的现场活动

## How People Usually Enter

很多协作关系并不是从正式“加入申请”开始，而是从这些活动场景进入:

- 先参加一次公开活动或线上分享
- 再围绕某个系列、某个研究方向或某个小模块发生协作
- 最后根据节奏，转为项目制或长期协作

## If You Want To Join

目前更适合的进入方式有四种:

1. 围绕某个项目系列参与一次清晰的协作
2. 围绕研究、写作或知识网络贡献一个小模块
3. 围绕工具、模板或发布结构参与搭建
4. 先从活动、沙龙和公开交流建立连接

更具体的参与说明见: [Collaboration](../collaboration/README.md)

# Why VIRTURA-Collective

日期：2026-03-16

这篇文章记录 `VIRTURA-Collective` 作为团队主仓被正式建立的原因。

## Background

在升级前，VIRTURA 的公开信息存在一个长期问题：

- `VIRTURA` 是团队名
- `SpacePort` 更像场所、平台或世界观入口
- 但很多团队介绍、作品入口、组织说明又被放进了 `SpacePort`

结果是，外部人第一次来到 GitHub 时，很容易先理解到一个“太空港 / 平台 / 世界观容器”，却没有先理解到：

- 这是一个团队
- 团队是谁
- 成员与作品在哪里
- 团队与工具、研究、发布之间是什么关系

## Core Judgment

这次调整的核心判断是：

`VIRTURA` 必须重新回到总品牌位置，而 `SpacePort` 退回它更适合的角色，也就是导览前台、入口大厅与生态地图。  

团队本体需要一个更清晰、更稳的名字来承接：

- 团队介绍
- 成员
- 作品
- 研究
- 合作

因此，`VIRTURA-Collective` 被正式采用。

## Why Not Keep SpacePort as the Main Team Name

`SpacePort` 适合表达：

- 接待
- 导航
- 空间感
- 轻度叙事包装

但它不适合做团队主身份，因为它天然更像一个场所，而不是一个创作共同体。

如果让它继续承担团队本体角色，系统会持续混淆：

- 团队是谁
- 平台是什么
- 工具是什么
- 发布出口是什么

## Why Collective

`Collective` 比 `Team` 更准确。

原因很直接：

- 它更适合去中心化协作
- 它更适合项目制参与
- 它不那么行政化
- 它更像艺术共同体而不是固定组织架构

这与 VIRTURA 当前真实的工作方式更加一致。

## New Relationship

正式关系被整理为：

- `portfolio`: 钱誉文个人入口
- `VIRTURA-Collective`: 团队主入口
- `VIRTURA-SpacePort`: 公共导览前台
- `VIRTURA-Newsroom`: 发布出口
- `SceneForge`: 场景工坊与 Digital Stage 实验仓

这让每个仓库第一次有了更清楚的角色分工。

## What This Repository Solves

`VIRTURA-Collective` 建立后，最直接解决的是三件事：

1. 团队身份不再寄居在 `SpacePort` 里
2. `organization` 与 `works` 不再在第一入口层割裂
3. 后续成员、作品、研究、合作都有了一个稳定主楼

## What Happens Next

这个仓库建立以后，不代表所有内容立即迁移完成。

更实际的顺序是：

1. 先建立角色和入口
2. 再迁核心内容
3. 再清理旧入口中的混合叙事
4. 最后让 `SpacePort / Newsroom / SceneForge` 回到各自角色

这也是为什么 `VIRTURA-Collective` 的第一版先是一个清晰的骨架，而不是一个堆满内容的新壳。

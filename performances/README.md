# Performances

这里保留 VIRTURA 的现场、舞台、展演与事件型项目入口。

## Why This Section Exists

VIRTURA 的作品形态不只是静态项目，也包括：

- 现场视觉演出
- 沉浸式空间事件
- 线上放映与发布
- 展览环境中的时间性作品

因此 `Performances` 作为独立入口存在，不再被完全混入一般作品页。

## Performance Types

- live performance
- exhibition event
- digital stage release
- site-specific presentation

## Why This Matters

把 `Performances` 独立出来的目的，是让团队的“发生型作品”拥有自己的入口逻辑，而不是全部被压平为一般作品列表。

它和 [Activities](../activities/README.md) 的区别是:

- `Performances` 更偏向作品发生的事件与场域
- `Activities` 更偏向团队公共活动、沙龙、工作坊与交流时间线

## Metadata Rule

后续所有展演或事件型项目，优先补这 3 类信息：

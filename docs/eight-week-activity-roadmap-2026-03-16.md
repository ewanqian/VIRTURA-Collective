# VIRTURA 未来 8 周活动路线图 2026-03-16

这份文档直接服务于三件事：

- 给 `lu.ma/VIRTURA` 提供未来 1-2 个月的真实排期
- 让 `VIRTURA-Collective/activities` 不只停留在活动说明，而有一条可持续推进的时间线
- 让 `VIRTURA-SpacePort/stations` 里的活动站点、展映入口和知识网络，能按节奏被反复激活

它不是纯脑暴，而是一份偏执行的工作稿。

---

## 一、当前编排原则

这轮排期沿用现在已经成立的三条主线：

- `Space Salon / 数字沙龙 / 交流客厅`
- `太空气球 / Baloon Live Space`
- `MOVA / Museum of Virtual Art`

同时穿插三类轻量节点：

- `Practice`
- `Concept Briefing`
- `Release / Archive Opening`

这样做的目的很明确：

1. 先让活动持续发生，而不是把所有压力都压在一次大活动上
2. 让不同角色的人都能找到进入方式
3. 让 GitHub 里的页面不只是资料库，而会被不断转成可参加、可观看、可回访的入口

---

## 二、未来 8 周的总体节奏

这 8 周不追求“每场都很重”，而追求节奏稳定：

- 周五晚上：更适合 `MOVA`、`Concept Brief`、`Release`
- 周六晚上：更适合 `Space Salon`、`太空气球`、`Practice`
- 关键判断：
  - `Space Salon` 负责社群交流与方法传播
  - `太空气球` 负责线上演出空间、音画实验与共测
  - `MOVA` 负责线上展映和轻量发布

## 五、每场活动为什么这样排

### 1. 先把“持续发生”做出来

`2026-03-21` 到 `2026-04-04` 这段，不要急着讲完整体系。

更重要的是先让外部感知到：

- 你们真的在测试
- 你们真的在交流
- 你们真的在发布

所以这段的主线是：

- `Alpha`
- `Vol.3`
- `MOVA 开幕`
- `Practice Night`

### 2. 再把“方法”讲出来

`2026-04-10` 到 `2026-04-25` 这段，更适合把知识和方法往前推。

原因是外部已经先见到发生中的节点，这时再讲：

- 数字情感容器
- 存档实践
- 知识网络怎么用
- 扫描到高斯

会比一上来就讲抽象主张更容易被接受。

### 3. 最后把“展映与迭代”拉通

`2026-05-01` 以后更适合做：

- 第二个线上小展
- 第二次 practice
- 第二轮 Beta 共测

因为这时第一波公共线已经形成回访理由，适合把人从“看一次”变成“继续跟着走”。

## 七、每场最少需要准备什么

为了防止排期看起来很多、实际落不下去，建议每类节点都只要求最少准备量。

| 节点类型 | 最少准备项 |
|---|---|
| `Space Salon` | 一个主题、一位主讲或主分享人、一条后续阅读入口 |
| `太空气球` | 一个测试目标、一段可播放内容、一种反馈方式 |
| `Practice` | 一个明确问题、一种带来材料的方式、一条后续联系入口 |
| `Concept Brief` | 一页核心文档、一个具体案例、一次 20-40 分钟结构 |
| `Release` | 一个真的能看的页面、一个明确开放时刻 |
| `MOVA` | 一条观看路线、一个开幕时刻、2-4 个入口页面 |

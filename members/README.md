# Members

这里整理 VIRTURA 当前公开可识别的成员结构与参与方式。

## Member Model

VIRTURA 不是一个固定编制团队，而是一个去中心化协作网络。

因此成员页要同时承接 3 种关系：

1. 持续协作的核心成员
2. 围绕单个系列或项目加入的协作者
3. 未来可能加入的项目制成员

## Participation Types

- core collaboration: 持续参与多个系列和方向
- project-based: 围绕某个项目或某个版本加入
- research/documentation: 围绕方法、文档、归档参与

## Role Clusters

当前最常出现的角色包括：

- visual art
- electronic music
- interaction
- stage and spatial design
- research and writing
- documentation and archiving

## Joining Context

如果未来补正式加入入口，最适合先记录的是：

- 你关注的方向
- 你更适合项目协作、研究协作还是长期协作
- 你可投入的时间
- 你已有的相关经验

## Current Rule

成员页优先记录公开可确认的协作关系，不急着制造过度确定的组织架构。

# VIRTURA 活动策划库 2026-03-16

> 注：这份文档保留为内部选题母表。当前对外活动命名以 `Space Salon`、`太空气球`、`MOVA`、`在制内容审计会`、`Digital Library / Digital Stage 公开说明` 这几条线为准，尽量不用更空、更像说明会的旧命名。

这份文档不是对外活动文案，而是给团队内部排期、选题和判断活动成熟度用的母表。

它的目标很简单：

- 先把 `VIRTURA-SpacePort` 目前已经成形的知识点、作品线和活动线压成一张可读地图
- 再把这些内容转成一组适合当前阶段的公共活动题库
- 最后帮助团队判断：什么适合立刻做，什么还应该继续积累

## 一、目前已经成形的内容地形

### 1. 作品主线

当前最适合被反复拿出来讲、看、演示和回访的作品线，主要是这三条：

- `Drop Flow`
  关键词：音画互动、现场版本、空间叙事、版本演进、线上线下切换
- `TIMER`
  关键词：时间结构、节奏控制、视觉组织、现场版本与规划版对照
- `Observation and Symbiosis`
  关键词：展览、数据景观、观察方法、扫描转译、公共活动与工作坊

### 2. 研究主线

`Collective/research` 当前已经公开展开的四条主线，可以直接转成活动母题：

- `Digital Library / Digital Stage`
  关注作品、演出、研究和档案如何进入公开前台
- `Spatial Creative Framework`
  关注空间感、观看路径、场景结构和跨媒介语言
- `Intellectual Equipment`
  关注知识如何从静态资料转成可调用、可学习、可部署的路径
- `Digital Emotion Container`
  关注情感锚点、空间记忆、扫描转译与可分享的数字空间

### 3. 知识网络主轴

`SpacePort/knowledge-network` 目前最清楚的 10 个知识领域如下：

- 空间计算 / 设备
- Blender 实时引擎
- 音画互动
- 3D 重建 / 可视化
- 网页端 / 发布
- 生产管线
- 角色体系
- 学习路线
- 现场表演 / 剧场艺术
- 我们的理论 / 后日谈

这些领域里，当前最适合拿来做公开活动的，不是“完整课程”，而是下列几种更真实的切口：

- 从零做一个小 Demo 的最短路径
- 一次现场作品是怎么被组织出来的
- 一个方法为什么值得保留下来
- 一个技术为什么对创作真的有用
- 一个项目为什么会卡住，以及团队怎么解决

### 4. 现有活动线

目前已经成立、而且可以继续扩展的公共活动与发布线，主要有三条：

- `Space Salon / 数字沙龙 / 交流客厅`
  更适合分享、漫谈、邀请嘉宾、小规模工作坊和方法传播
- `太空气球 / Baloon Live Space`
  更适合线上演出空间测试、音画实验、公开测试、直播性活动和后续线上线下演出节点
- `MOVA / Museum of Virtual Art`
  更适合线上展映、作品观看室、library 转展览的快速发布，以及档案型展览入口

## 二、活动设计原则

### 1. 一场活动只回答一个问题

不要把一场活动写成“团队完整世界观说明会”。

更好的做法是每一场只回答一个明确问题，例如：

- 一次现场演出为什么值得被存档
- 为什么音频驱动视觉对你们很重要
- 什么叫数字情感容器
- 为什么知识网络不能只是一堆文档

### 2. 先讲真实过程，再讲抽象主张

现在最打动人的，不是“我们有体系”，而是：

- 你们真的做过什么
- 你们怎么做出来的
- 你们在哪些地方卡住过
- 这些经验为什么值得继续开放分享

### 3. 每场活动至少挂一个真实入口

每场活动至少应该能挂住下面三种入口中的一种：

- 一个作品入口
- 一个知识网络入口
- 一个后续活动入口

这样活动结束后，读者不会只记住一个海报标题，而能找到继续看的地方。

### 4. 公共活动优先做“可进入”，不是做“很满”

目前很多知识页的第二层还在补充，所以公开活动更适合做：

- 轻量分享
- 方法漫谈
- 公开测试
- 小范围工作坊

不适合一上来就做：

- 宣称覆盖完整体系的大课
- 过度产品化的发布会
- 需要大量稳定教学材料支撑的重型训练营

### 5. 让 Luma 承担“长期订阅”，让 GitHub 承担“继续深读”

活动的基本闭环建议固定为：

1. 在 `lu.ma/VIRTURA` 订阅后续活动
2. 在 `VIRTURA-Collective` 看团队、作品、活动和合作入口
3. 在 `VIRTURA-SpacePort` 深读知识网络、项目档案和活动归档

### 6. 不是所有日历节点都必须是“见面活动”

对 VIRTURA 来说，Luma 不应该只用来挂线下聚会和线上会议。

只要有明确的公开开始时间、观看窗口、申请入口或测试时段，下面这些内容都可以进入日历：

- `practice / 共练 / 开放排练`
- `concept briefing / 概念短讲 / 方法说明`
- `public test / alpha / beta`
- `场景内容上新`
- `作品档案或展映页面开放`
- `MOVA 线上展映开启`

## 三、建议保留的 5 类活动结构

### A. 公开认识型

适合第一次认识 VIRTURA 的人。

特点：

- 难度低
- 门槛低
- 不要求参与者有很强的技术背景
- 重点是建立兴趣与信任

优先放在：

- `Space Salon`

### B. 创作者方法型

适合成长中的创作者、研究同好、对方法和工作流感兴趣的人。

特点：

- 讲方法，但不装完整课程
- 以真实项目片段为例
- 能让人带着一个清楚的小收获离开

优先放在：

- `Space Salon`
- 小范围工作坊

### C. 实验测试型

适合音乐人、视觉创作者、开发者和愿意一起测试空间的人。

特点：

- 更强调实时性和参与感
- 更适合线上线下联动
- 能为后续作品和平台迭代收集反馈

优先放在：

- `太空气球 / Baloon Live Space`

### D. 归档复盘型

适合潜在合作方、策展方、机构伙伴，以及希望更深入理解团队方法的人。

特点：

- 讲清项目如何组织
- 讲清版本、角色与方法
- 讲清从现场到存档的转换过程

优先放在：

- `Space Salon`
- 邀请制闭门分享

### E. 发布展映型

适合把 GitHub 里的作品、档案、library 和研究页转成更可观看的公共入口。

特点：

- 不一定需要很重的直播或线下组织
- 更适合和日历、开幕时段、观看窗口配合
- 适合快速发布，也适合长期留存

优先放在：

- `MOVA / Museum of Virtual Art`
- `lu.ma/VIRTURA` 的上线节点

## 五、最值得优先启动的一批题目

如果只按“现在最该做、又最容易做好”的标准来排，建议优先从这 12 个开始：

1. 从一次活动认识 VIRTURA：我们为什么持续做空间茶话会
2. 我们现在最重要的三条作品线：Drop Flow、TIMER、Observation and Symbiosis
3. 一次线下现场，怎样变成可回看的数字档案
4. 音频如何驱动视觉：从最简单的节奏反应讲起
5. 从扫描到高斯：如何把现实里的东西带回数字空间
6. 网页端不是降级方案：如何把空间内容发布到公开前台
7. 知识网络怎么做成“能用”的，而不是“能看”的
8. 太空气球 Alpha 共测夜：一起测试线上演出空间
9. 从库到展：VIRTURA 数字空间研究摘编
10. 场景上新：本周发布一个新的页面、房间或观看入口

这一批题的共同点是：

- 都能直接挂住你们已经存在的作品或文档
- 都不用假装自己已经有一整套成熟课程
- 都能自然引向后续订阅、归档和合作

## 六、建议暂时不要先做的题型

下面这几类活动现在不建议优先做：

### 1. “完整体系大课”

原因：

- 当前知识网络第二层还在补
- 做太满容易暴露材料不均衡
- 听众也容易疲劳

### 2. “正式产品发布会”

原因：

- 你们更适合做持续测试和公开发生
- 太早把活动写成发布会，反而会让人误以为你们需要装成熟

### 3. “纯技术参数型讲座”

原因：

- VIRTURA 的优势不是纯参数堆砌
- 你们真正有辨识度的是：空间、作品、方法、档案和现场之间的连接

### 4. “过强的加入导向活动”

原因：

- 不是每个人都想加入
- 很多人只是想先旁听、关注、参加活动、慢慢形成关系
- 公共活动页应该优先服务这种更自然的进入方式

### 5. “没有开始时间的假发布”

原因：

- Luma 适合承接有明确开始时间、申请窗口或观看窗口的节点
- 如果只是页面悄悄更新，不值得单独发一个日历节点
- 但如果它是一次开幕、上线、开放申请或公共测试，就可以被当成日历内容

## 八、后续可以继续补的字段

如果你们后面想把这份题库继续做成更可执行的排期母表，建议再给每个题目补四个字段：

- `主持人 / 适合邀请谁`
- `需要准备的素材`
- `是否适合录制 / 二次发布`
- `活动结束后的推荐阅读`

这四项一补，活动库就会从“点子清单”升级成真正能排起来的节目单。
